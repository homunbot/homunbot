## 2. Analisi Competitor

### OpenClaw (TypeScript/Node.js)

**Punti di forza chiave:**
- **Auth profile rotation**: multipli profili per provider con cooldown/failover
- **50+ tool**: browser Playwright, canvas A2UI, Discord/Slack actions, sessions
- **15+ canali**: WhatsApp, Telegram, Slack, Discord, Signal, iMessage, Teams, Matrix, etc.
- **DM Pairing**: senders sconosciuti ricevono codice di approvazione
- **Session branching**: conversazioni divergenti con merge
- **Token counting**: usage tracking con grafici nel Web UI
- **Lobster workflows**: workflow multi-turn con context isolation
- **Canvas (A2UI)**: workspace visivo agent-driven
- **macOS/iOS/Android app**: companion app con Voice Wake e Talk Mode
- **i18n**: inglese + cinese

**Debolezze vs Homun:**
- No MCP client nativo (usa mcporter bridge)
- No exfiltration filter
- Node.js runtime (~1GB RAM, 500ms+ startup)
- Dipendenza da Playwright per browser

### ZeroClaw (Rust)

**Punti di forza chiave:**
- **25+ canali**: DingTalk, Lark, Signal, Matrix, IRC, etc.
- **Hybrid memory**: vector 0.7 + keyword 0.3 scoring, SQLite BLOB embeddings
- **AIEOS v1.1**: formato identita strutturato (JSON schema con OCEAN traits)
- **Security hardened**: Landlock, Bubblewrap, E-Stop 4 livelli, OTP
- **Observability**: Prometheus, OpenTelemetry, runtime tracing
- **Edge computing**: firmware ESP32/STM32/RPi (<5MB RAM, <10ms startup)
- **38+ tool**: Composio (100+ SaaS), git, HTTP, hardware introspection
- **Session compaction**: auto-summarization quando >50 messaggi

**Debolezze vs Homun:**
- No MCP client
- Web UI minimale (solo gateway basic)
- No ClawHub/skill marketplace
- Niente skill hot-reload
- Codebase molto piu grande e complesso

## 4. Vantaggi Competitivi di Homun

Cose che Homun fa gia bene e che i competitor NON hanno o fanno peggio:

| Feature | Dettaglio |
|---------|-----------|
| **MCP client nativo** | First-class, non un bridge. Ne OpenClaw ne ZeroClaw |
| **Browser CDP diretto** | Senza Playwright/Node.js. Single binary |
| **Exfiltration filter** | OpenClaw non ce l'ha |
| **Web UI ricca** | 10+ pagine. ZeroClaw ha solo gateway basic |
| **Skill ecosystem completo** | ClawHub + OpenSkills + hot-reload + security audit |
| **Single binary Rust** | ~50MB, no runtime, <10ms startup |
| **Vault con OS Keychain** | AES-256-GCM + keyring nativo |
| **XML fallback automatico** | Supporta modelli senza function calling |
| **Prompt modulare** | Sezioni componibili, skip per mode |
| **3-layer shell security** | Deny patterns + regex + risky commands |

## 5. Bug Architetturali Corretti (PR #21)

Data fix: 2026-03-03

| Bug | Root Cause | Fix |
|-----|-----------|-----|
| Routing rules invisibili in native mode | `ToolsSection` early-return quando `ctx.tools` vuoto | `registered_tool_names` sempre popolato |
| Browser description troppo densa | ~1036 chars in tool JSON schema | Ridotto a ~200 chars, workflow in system prompt |
| XML fallback contraddittorio | Append a prompt "No tools" | Rebuild da zero |
| Snapshot troppo grandi | Nessun limite | compact: true, limit: 75, truncation notice |
| "Reflect on results" inutile | Messaggio user fittizio ogni iterazione | Rimosso |
| Nessun log context size | - | chars + messaggi loggati per chiamata |

## 7. Feature Flags (Cargo.toml)

```
default    = cli, web-ui, file-tools, shell-tool, web-tools, cron-tool,
             message-tool, vault-2fa, mcp
gateway    = default + channels + local-embeddings
full       = gateway + browser + vault-2fa
browser    = chromiumoxide
local-embeddings = fastembed + usearch
mcp        = rmcp
channel-telegram, channel-discord, channel-whatsapp, channel-email

```

Build variants:
- `cargo build --release` — Core (~15MB)
- `cargo build --release --features gateway` — Gateway (~40MB)
- `cargo build --release --features full` — Full (~50MB)
